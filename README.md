
# Automated Data Pipeline: Decentralized Inventory Management #

## Project Overview ##

Maintaining data integrity across a massive, decentralized network of physical assets is a major operational challenge. This project constructs an automated machine learning pipeline using Python to clean, validate, and standardize chaotic input data from Airbnb's decentralized lodging network.

##  Tools & Technologies ##

* **Language:** Python
* **Libraries:** Pandas, Scikit-learn, MLflow, Weights & Biases
* **Methodology:** Data Pipeline Engineering, Automated Data Wrangling

##  The Problem ##

Operations managers cannot optimize physical logistics if the underlying data is broken. Decentralized networks naturally produce inconsistent, missing, and messy data points. The objective of this project is to:

1. Build a robust pipeline that automatically ingests and processes raw data.
2. Handle missing values, standardize formats, and validate inputs without requiring manual, line-by-line auditing.
3. Ensure the final dataset is clean, reliable, and ready for advanced operational modeling.

##  Business Impact & Key Findings ##

* **Ensuring Data Integrity:**

By automating the cleaning process, this pipeline eliminates human error and ensures that downstream operational dashboards reflect accurate reality.

* **Automated Auditing:**

Just as physical freight must be audited against purchase orders, this pipeline acts as an automated auditor for digital inputs, saving countless hours of manual review.

* **Scalable Data Architecture:**

A well-engineered pipeline allows an organization to ingest exponentially larger datasets without overwhelming the system or the analytics team.
